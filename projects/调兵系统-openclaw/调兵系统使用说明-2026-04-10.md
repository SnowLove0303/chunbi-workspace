# 调兵系统使用说明

日期：2026-04-10

## 一、这份说明是干什么的

这份文档是调兵系统的实际使用手册。

重点不是讲架构理念，而是讲：

- 命令怎么用
- 各功能分别做什么
- 什么时候该用哪一个入口
- 参数该怎么写
- 常见调用方式有哪些

当前主入口：

```powershell
F:\codex\codex\tools\assistant-hub.ps1
```

CMD 包装入口：

```cmd
F:\codex\codex\tools\assistant-hub.cmd
```

## 二、先记住三条总原则

### 1. 不同入口代表不同调度深度

- `status` / `list-*` 是查看状态
- `recommend` / `auto` 是自动选型
- `swarm` 是并行群呼
- `commander` 是总指挥带队
- `openclaw-commander` 是任务直接扔给 OpenClaw 总指挥
- `formation ...` 是强制走指定编队

### 2. 先轻后重

推荐默认顺序：

1. 先看状态
2. 再看 roster
3. 小任务先单兵或轻编队
4. 需要对比、分工、复核时再上 swarm / commander / heavy

### 3. 能点名就尽量点名

如果你已经知道要用哪个 agent，就直接点名，不要盲调。

例如：

```powershell
F:\codex\codex\tools\assistant-hub.ps1 commander codex "openclaw@codex-reviewer,copaw@default,opencode@writer" "你的任务"
```

## 四、查看类命令怎么用

### 1. `help`

作用：查看当前支持的全部命令

```powershell
F:\codex\codex\tools\assistant-hub.ps1 help
```

### 2. `status`

作用：看三兵当前总体状态（CoPaw / OpenClaw / OpenCode）

```powershell
F:\codex\codex\tools\assistant-hub.ps1 status
```

### 3. `list-copaw-agents`

作用：列出当前 CoPaw agent roster

```powershell
F:\codex\codex\tools\assistant-hub.ps1 list-copaw-agents
```

### 4. `list-openclaw-agents`

作用：列出当前 OpenClaw agent roster

```powershell
F:\codex\codex\tools\assistant-hub.ps1 list-openclaw-agents
```

### 5. `list-opencode-agents`

作用：列出当前 OpenCode 角色 roster

```powershell
F:\codex\codex\tools\assistant-hub.ps1 list-opencode-agents
```

## 五、直通原生命令

### `copaw [args...]` / `openclaw [args...]` / `opencode [args...]`

直接透传给对应 CLI。

### `copaw-task <agent-id> <message...>`

把任务直接发给指定 CoPaw agent。

```powershell
F:\codex\codex\tools\assistant-hub.ps1 copaw-task default "用一句话总结这件事"
```

## 六、场景预检查

### `scenario check`

快速检查三兵整体状态。

```powershell
F:\codex\codex\tools\assistant-hub.ps1 scenario check
```

## 七、自动选型

### `recommend <message...>`

只判断该用哪种编队，不执行。

### `auto <message...>`

自动选型并执行。

## 八、并行群呼

### `swarm <message...>`

把同一条任务并行发给三兵，回收三路结果。适合短平快对比，不适合重型协作。

```powershell
F:\codex\codex\tools\assistant-hub.ps1 swarm "你的任务"
```

## 九、总指挥模式

### `commander <commander> [workers] <message...>`

由 commander 统筹派工并收口。

```powershell
# 默认 workers
F:\codex\codex\tools\assistant-hub.ps1 commander codex "你的任务"

# 指定 workers
F:\codex\codex\tools\assistant-hub.ps1 commander codex "copaw,openclaw,opencode" "你的任务"

# 点名
F:\codex\codex\tools\assistant-hub.ps1 commander codex "openclaw@codex-reviewer,copaw@default,opencode@writer" "你的任务"
```

### `openclaw-commander <message...>`

直接让 OpenClaw 当总指挥，不再走本地脚本。

```powershell
F:\codex\codex\tools\assistant-hub.ps1 openclaw-commander "你的任务"
```

## 十、强制编队

### `formation <name> <message...>`

强制走预设编队。

可选：`mixed` / `research` / `coding` / `ops` / `debug` / `content` / `publish` / `openclaw-heavy`

```powershell
F:\codex\codex\tools\assistant-hub.ps1 formation coding "设计一个待办解析器"
F:\codex\codex\tools\assistant-hub.ps1 formation openclaw-heavy "重型 OpenClaw 团队预检"
```

## 十一、OpenCode 角色

### `opencode-role <role> <message...>`

直接调用 OpenCode 某个角色：`exact` / `writer` / `coder` / `reviewer` / `default`

```powershell
F:\codex\codex\tools\assistant-hub.ps1 opencode-role writer "写一句中文"
```

## 十二、ClawTeam 重型团队

- `clawteam-smoke` — 冒烟测试
- `clawteam-template-list` — 列出模板
- `clawteam-template-show <template>` — 查看模板详情
- `clawteam-launch <template> <goal>` — 拉起团队
- `clawteam-overview` — 团队概况
- `clawteam-board [team] [port]` — Web 看板（默认 8080）
- `clawteam-tasks <team>` — 任务列表
- `clawteam-inbox-log <team> [limit]` — 消息日志

## 十三、worker 写法

```
copaw,openclaw,opencode              # 三兵各一
openclaw:2,copaw,opencode          # OpenClaw 两个，其余各一
openclaw@codex-reviewer,copaw@default,opencode@writer  # 点名
```

## 十四、推荐选型顺序

1. `status` — 开工前体检
2. `recommend` / `auto` — 让系统判断
3. `swarm` — 想对比
4. `commander codex` — 想稳定交付
5. `openclaw-commander` — 让 OpenClaw 自己当总指挥
6. `formation openclaw-heavy` / `clawteam-*` — 重型团队化

## 十五、已知注意事项

- `recommend` 有偶发 SIGKILL（CLI WS Bug #47133）
- `openclaw-commander` 偶发带历史上下文残留
- 点名前先看 roster，不要凭印象
- OpenCode web 需手动 `opencode web` 启动

## 十六、一句话总结

按任务复杂度逐级上强度：状态查看 → 自动选型 → 并行对比 → Codex 总指挥 → OpenClaw 直达总指挥 → 重型 ClawTeam。
